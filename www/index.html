A Pattern Matching Compiler for Multiple Target Languages

Jtom: a (Java) To One Matching compiler

TOM is a language extension which adds pattern matching facilities to an existing imperative language.
In the current version, TOM supports C, Java and Eiffel as native language.

Principles

The tool does not impose a fixed term representation, rather it accepts implementation of terms (or term like data types) of yet existing applications and it permits to define and execute rewriting rules upon those types.
The main design of the implementation can be decomposed into several parts:
  • A parsing phase that generates Abstract Syntax Trees (using ATerms)
  • An expansion phase that performs syntactic sugar replacement
  • A type-checking phase that performs type inference
  • A transformation phase that performs program transformation
  • A compilation phase that transforms the AST into an abstract program
  • A generation phase that transform the abstract program into the target language
