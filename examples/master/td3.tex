\section{Arithmétique  de Peano}

On considère le système de réécriture permettant de calculer la somme
d'entiers de Peano:
%
\begin{itemize}
\item[] $x + zero \rightarrow x$
\item[] $x + suc(y) \rightarrow suc(x + y)$
%% \item[] $x * zero \rightarrow zero$
%% \item[] $x * suc(y) \rightarrow x + (x*y)$
\end{itemize}
%
L'opérateur $+$ est associatif et commutatif.

En utilisant le système obtenu calculer les sommes $0+2$ et $0*2$. 

\begin{versionProf}
$0+2 \longrightarrow 2$
\end{versionProf}

\section{Polynômes}

On considère le système de réécriture permettant de calculer la
dérivée de tout polynôme en sachant que:
\begin{itemize}
\item[] $deriv(n,X) \rightarrow 0$
\item[] $deriv(m*X^n,X) \rightarrow m*n*X^{n-1}$
\item[] $deriv(P1+P2,X) \rightarrow deriv(P1,X) + deriv(P2,X)$
%% \item[] $deriv(P1*P2,X) \rightarrow deriv(P1,X) * P2 + deriv(P2,X) * P1$
\end{itemize}
%
Nous avons vu que le résultat obtenu en appliquant ce système de
réécriture n'est pas en forme canonique.

Proposez un système de réécriture équationnelle permettant de résoudre
ce problème. Précisez les règles de réécriture utilisées ainsi que les
équations des différents symboles.

\begin{versionProf}
Si l'opérateur $+$ est associatif et commutatif:
%
\begin{itemize}
\item[] $simplify(m_1 X^n +_{AC} m_2 X^n) \rightarrow (m_1+_{AC}m_2) X^n$
\end{itemize}
%

Si l'opérateur $+$ est associatif:
\begin{itemize}
\item[] $simplify(m_1 X^n +_{A} P +_{A} m_2 X^n) \rightarrow (m_1+_{A}m_2) X^n+_{A} P$
\item[] $simplify(m_1 X^{n_1} +_{A} P +_{A} m_2 X^{n_2}) \rightarrow m_2 X^{n_2} +_{A} m_1 X^{n_1} +_{A} P$
  ~~~~~~if $n_1 < n_2$
\end{itemize}
\end{versionProf}

\section{Vérification de protocoles cryptographiques}

On souhaite analyser le protocole NSPK proposé en '78 par Needham et
Schroeder.  Le protocole est utilisé pour établir une authentification
mutuelle entre deux agents communicant via un réseau qui n'est pas
sécurisé.

\newcommand{\encMes}[2]{{\{#1\}}_{K(#2)}}
\newcommand{\nonce}[1]{{N}_{#1}}
\begin{tabbing}
indent \=  3.~~~ \= $A \rightarrow B$: \= \kill
\>  1. \> $A \rightarrow B$: \> $\encMes{\nonce{A},A}{B}$
\\
\>  2. \> $B \rightarrow A$: \> $\encMes{\nonce{A},\nonce{B}}{A}$
\\
\>  3. \> $A \rightarrow B$: \> $\encMes{\nonce{B}}{B}$
\end{tabbing}

On souhaite écrire le système de réécriture permettant de vérifier le
protocole en présence d'un intrus qui peut intercepter tous les
messages qui passent par le réseau (à l'exception des messages qu'il a
engendrés) et les mémorise ou les décrypte (si encryptés avec sa clé).
Il peut également utiliser les nonces obtenus pour générer de faux
messages qui sont envoyés à tous les agents.

On souhaite vérifier:
\begin{itemize}
\item l'authenticité du répondeur~: si un initiateur $x$ a établie une
  session avec un répondeur $y$, alors $y$ a vraiment participé au
  protocole.
%
\item l'authenticité de l'initiateur~: si un répondeur $y$ a établie
  une session avec un initiateur $x$, alors l'initiateur a également
  commité avec $y$.
\end{itemize}

\section{Prouveur de prédicats}

On souhaite écrire un système de réécriture permettant de prouver des
propositions construites en utilisant les symboles \ANDh (et), \ORh
(ou), \IMPLh (implication), \EQUIVh (équivalence), \NOTh
(négation). 

Les conjectures à prouver sont décrites par des sequents qui consiste
en un certain nombre d'hypothèses $Hyps$ et un but $P$ et sont représentés
sous la forme:
\begin{center}
$Hyps \vdash P$
\end{center}
Les règles d'inference utilisées pour prouver un  sequent ont la forme:
\[\frac{S_1  S_2 ... S_n}
        %
        {S}
\]
où les sequents $S_1, S_2, ... S_n $ sont appelés les
\textit{antecedents} de la règle et le sequent \textit{S} est appelé
le \textit{consequent}. Ce type de règle indique que la preuve du
consequent peut être réduite aux preuves de ses antecedents. On
utilise par la suite la représentation suivante des règles
d'inference:
%
\begin{center}
\begin{tabular}{|c|c|c|}\hline 
  & Antecedents &             Consequent \\ \hline \hline
  RuleName & $S_1,  S_2, ... S_n $ & S \\ \hline 
\end{tabular}
\end{center}
%% où $S_i$ sont les antecedents qui peuvent être des sequents de la
%% forme $H \vdash P$. 
Les règles sans sequents dans les
antecedents sont des axiomes.

On considère les règles d'inférence:

//--------------------------------------------------------------------------------------
//AND-rules 
  [] H |- ^(P && Q) |=> R     =>  H |- ^Q |=> R  :: H |- ^P |=> R          end
  [] H |- ^(P && Q)           =>  H |- P |=> ^Q          end
  [] H |-  (P && Q) |=> R     =>  H |- P |=> (Q |=> R)           end
  [] H |-  (P && Q)           =>  H |- Q :: H |- P          end
//--------------------------------------------------------------------------------------
//OR-rules
  [] H |- ^(P or Q) |=> R     =>  H |- ^P |=> (^Q |=> R)           end
  [] H |- ^(P or Q)           =>  H |- ^Q :: H |- ^P          end
  [] H |-  (P or Q) |=> R     =>  H |- Q |=> R :: H |- P |=> R     end
  [] H |-  (P or Q)           =>  H |- ^P |=> Q               end
//--------------------------------------------------------------------------------------
//IMP-rules
  [] H |- ^(P |=> Q) |=> R   =>  H |- P |=> (^Q |=> R)           end
  [] H |- ^(P |=> Q)         =>  H |- ^Q :: H |- P               end
  [] H |-  (P |=> Q) |=> R   =>  H |- Q |=> R :: H |- ^P |=> R     end 
  [] H |-  (Pp |=> Q)        =>  H && Pp |- Q               end
  [] H |-  (^Pp |=> Q)       =>  H && ^Pp |- Q               end
//--------------------------------------------------------------------------------------
//EQV-rules
  [] H |- ^(P <=> Q) |=> R    => S1 :: S2
         where S1 := () H |- P |=> (^Q |=> R)
         where S2 := () H |- ^P |=> (Q |=> R)          end
  [] H |- ^(P <=> Q)          => S1 :: S2
         where S1 := () H |- P |=> ^Q 
         where S2 := () H |- ^Q |=> P         end
  [] H |-  (P <=> Q) |=> R    => S1 :: S2
         where S1 := () H |- P |=> (Q |=> R)
         where S2 := () H |- ^P |=> (^Q |=> R)          end
  [] H |-  (P <=> Q)          => S1 :: S2
         where S1 := () H |- P |=> Q
         where S2 := () H |- Q |=> P         end
//--------------------------------------------------------------------------------------
//NOT-rules
  [] H |-  ^^P |=> R     => H |- P |=> R         end
  [] H |-  ^^P           => H |- P         end
//--------------------------------------------------------------------------------------
//AXM-axioms
  [] H && ^P |- P |=> Q    => $         end
  [] H && P  |- ^P |=> Q   => $         end
  [] H && P  |- P          => $         end
  [] H && P && ^P |- Q     => $          end
//----------------------------------------------------------------------------
//AXM-aux
  [] H && R |- P |=> R            => $          end
  [] H && ^Q  |- P |=> (Q |=> R)  => $          end
  [] H && Q  |- P |=> (^Q |=> R)  => $          end
  [] H |- P     |=> P             => $          end
  [] H |- P && R |=> R            => $          end
//-----------------------------------------------------------------------------
//AND-aux
  [] H |- P && a && (a |=>  b) |=> R     =>  H |- P && a && b |=> R      end
//--------------------------------------------------------------------------------------
//TRUE-rules
  [] H |- ^TRUE |=> R      => $              end
  [] H |- ^TRUE            => H |- FALSE          end
  [] H |-  TRUE |=> R      => H |- R          end
  [] H |-  TRUE            => $              end
//--------------------------------------------------------------------------------------
//FALSE-rules
  [] H |- ^FALSE |=> R      => H |- R         end
  [] H |- ^FALSE            => $              end
  [] H |-  FALSE |=> R      => $               end
//--------------------------------------------------------------------------------------
//STOP-rules
  [] H |- Pp      => H |- ^Pp |=> FALSE
              if neq_Prop(Pp,FALSE)          end
  [] H |- ^Pp     => H |- ^^Pp |=> FALSE
              if neq_Prop(Pp,FALSE)          end
//--------------------------------------------------------------------------------------
//Built-in
     [] $ :: S           => S                   end
//hypothese unique
     [] R && R && P |- Q      => R && P |- Q         end
\end{verbatim}
